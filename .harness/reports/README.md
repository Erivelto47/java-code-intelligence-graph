# Reports

Reports are post-execution records produced after a task or phase is completed.

They capture facts about what the Executor CLI actually did: files created,
files changed, validations run, tests executed or skipped, risks found, and the
next suggested step.

Real or temporary execution reports must stay outside this repository, under:

```text
/Users/eriveltomuller/Documents/GitHub/reports
```

The `.harness/reports/` directory stores report conventions and templates. It
does not necessarily store real execution reports.

Reports are generated by the Executor CLI and returned to the Planner/Reviewer
so the next decision is based on observed execution state.

Reports must not invent success. Failures, limitations, skipped checks, and
pending work must be recorded explicitly.
