# Validations

Validations registram checks esperados para uma fase. Elas devem combinar
comandos Git, build/testes e verificacoes especificas do contrato em andamento.

Use `validation-checklist.template.md` como base. Checks nao executados devem
ser justificados no report.

## Pacotes versionados

- `phase-4-2-java-decision-unresolved-early-return.validation.md`: checklist de
  validacao esperado para a execucao futura da Fase 4.2.
