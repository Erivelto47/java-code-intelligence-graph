# Handoffs

Handoffs preservam contexto entre fases, conversas, roles, ferramentas ou
operadores.

Eles devem ser objetivos e suficientes para que outro papel continue a execucao
sem depender de memoria de chat. Um handoff nao substitui o report:

```text
Report = registro factual de uma execucao concluida.
Handoff = contexto condensado para continuidade.
```

Use `handoff.template.md` quando houver transferencia entre ChatGPT Web
Architect, Codex CLI Executor e Human Reviewer, ou quando uma fase terminar
deixando proxima acao clara.

Handoffs temporarios devem evitar ruido no repositorio. Handoffs versionados
devem representar contexto estavel ou decisao operacional relevante.
