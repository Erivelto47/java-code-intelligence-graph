## Objective

Add conservative Java Decision Trace support for direct block `if/else`
branches where one branch is a direct literal throw and the other branch is a
simple direct return, represented as `IF_ELSE_CONDITION`.

## Context

Phase 4.3 supports narrow block `if/else` return/return extraction. This phase
extends only that narrow family to mixed throw/return branches while preserving
existing if/throw, single-line if/throw, early-return and return/return behavior.

## Branch strategy

Use the current working branch. Do not branch from `master`, merge to `master`
or push automatically.

## Scope

- Java source-text Decision Trace extraction under
  `com.codeatlas.adapter.java.source.decision`.
- Direct block `if/else` with one literal `throw new SomeException("message")`
  branch and one simple direct `return` branch.
- Optional symmetrical `return`/`throw` support if it remains simple.
- Fixture `examples/phase-4-decision-trace/07-if-else-throw-return-branches`.
- Exact fixture comparisons for `decisions.json`, `decisions.md` and
  `decisions.mmd`.
