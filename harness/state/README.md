# State

State registra uma fotografia leve da execucao atual. Ele deve ajudar agentes e
revisores a entender branch, fase, objetivo ativo, validacoes recentes e
proxima acao recomendada.

Use `current-state.template.md` para criar um estado quando houver necessidade
de preservar continuidade entre sessoes ou fases.

O state nao substitui report de fase nem historico Git. Ele deve permanecer
curto e objetivo.
