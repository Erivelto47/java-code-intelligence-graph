# State

State registra uma fotografia leve da execucao atual. Ele deve ajudar roles e
revisores humanos a entender branch, fase, objetivo ativo, validacoes recentes
e proxima acao recomendada.

Use `current-state.template.md` para criar um estado quando houver necessidade
de preservar continuidade entre sessoes ou fases. Use
`current-state.example.md` apenas como exemplo versionado.

O state nao substitui report de fase nem historico Git. Ele deve permanecer
curto e objetivo. Estado real volatil ainda nao deve ser versionado sem uma
decisao explicita.
